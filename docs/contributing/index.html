Contributing

Welcome

Contributions to BibleProject open source repositories are welcome. This page covers the basics of how to report issues, propose changes, and submit pull requests.

Reporting issues

Found a bug or have a question? Open an issue on the relevant GitHub repository:

  1. Navigate to the repo on GitHub
  2. Click Issues → New issue
  3. Use a clear, descriptive title
  4. Include steps to reproduce (for bugs) or a clear description (for feature requests)

Submitting pull requests

1. Fork the repository

Click Fork on the GitHub repository page to create your own copy.

2. Clone your fork

git clone https://github.com/your-username/repo-name.git
cd repo-name

3. Create a branch

git checkout -b feat/your-feature-name

Use a descriptive prefix:

  • feat/ — new feature
  • fix/ — bug fix
  • docs/ — documentation changes
  • chore/ — maintenance (dependency updates, build config)

4. Make your changes

  • Keep changes focused — one logical change per pull request
  • Follow the coding conventions already present in the project
  • Update documentation if your change affects public-facing behavior

5. Open a pull request

Push your branch and open a PR against the main branch of the upstream repository. Fill in the PR template — describe what changed and why.

Code of conduct

Be respectful and constructive. BibleProject open source projects follow a standard contributor code of conduct. In short: treat everyone with kindness.

Questions? Open an issue or reach out via the BibleProject website.
